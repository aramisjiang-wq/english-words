## 一、总览

本次样式重构的目标是：在不改变业务逻辑的前提下，降低样式维护成本，提高一致性，并为后续主题化（暗黑模式）预留扩展点。  
改造策略采用“渐进式收口”：先补 token，再把关键交互页面从内联样式迁移到 class 体系，最后清理散落的动态样式写法。

## 二、改造明细

### 2.1 设计变量与基础能力

- 在 `index.html` 中补充并启用核心 token：
  - `--color-*`（主色/成功/警告/危险/文字/边框）
  - `--focus-ring`
  - `--radius-sm`
- 新增统一焦点样式：`button/input/select/textarea:focus-visible`。
- 按钮体系统一到语义类：`btn-primary/success/danger/warning/secondary/violet/pink`。

### 2.2 会话页面样式重构（测验/练习/听力/拼写）

- 抽离模板到 `js/session-renderers.js`，替换大量内联样式模板：
  - `buildQuizQuestionHtml`
  - `buildPracticeWordHtml`
  - `buildListeningWordHtml`
  - `buildSpellingWordHtml`
  - `buildCompletionHtml`
- 在 `index.html` 新增会话类：
  - 布局：`.session-card`, `.session-actions`, `.session-input-wrap`
  - 内容：`.session-title`, `.session-phonetic`, `.session-example-box`
  - 音频区：`.session-audio-box`, `.session-word-center`
  - 反馈区：`.session-feedback` + `warning/error/success/info`

### 2.3 交互状态样式收口

- `toggleBatchMode` 从直接改 `style.background` 改为 class 切换：
  - 选中模式：`btn-danger`
  - 默认模式：`btn-warning`
- 高级筛选显示隐藏改为 class 逻辑：
  - `hidden` 类 + `classList.toggle('hidden')`
- 成就通知从 `style.cssText` 改为 class：
  - `.achievement-toast` 及子类。

### 2.4 统计与设置区域收口

- 设置弹窗模板去内联，改为 `settings-*` 类体系：
  - `.settings-panel`, `.settings-block`, `.settings-label`, `.settings-select` 等。
- 统计区域标题与空状态去内联：
  - `.analytics-title-inline`
  - `.analytics-empty`
- 进度区“今日需复习”去内联：
  - `.review-meta`
  - `.review-count`

## 三、当前效果与收益

- 样式可维护性显著提升：样式集中在 CSS 类，JS 主要负责结构与状态。
- 视觉一致性提升：按钮和反馈语义统一，交互态可预测。
- 后续迭代风险降低：新增功能时优先复用类，不必复制内联样式块。
- 兼容现有业务：未改核心学习逻辑、数据结构与功能入口。

## 四、已知保留项（合理）

以下内联样式仍保留，属于“运行时动态值”，保留是合理的：

- `mainContent.style.display = 'block'`（页面行为控制）
- 进度段 `style="flex: ${value}"`（数据驱动比例）
- 图表 Canvas 绘制过程中的动态样式（运行时绘图需要）

## 五、下一步建议（暗黑模式）

建议按以下顺序接入暗黑模式，改造成本最低：

1. 在 `:root` 完成 token 全量覆盖（仍有少量硬编码可继续替换）。
2. 增加 `.theme-dark`（或 `data-theme="dark"`）并覆盖 token。
3. 本地存储主题偏好（`localStorage.theme`）。
4. 增加设置开关并在加载时应用主题。
5. 优先验证高频区域：
   - 单词卡片
   - 会话弹窗
   - 图表区（坐标文字和对比度）

## 六、总结

本轮重构已完成“从内联样式到类体系”的主体迁移，前端样式进入可扩展状态。  
后续若继续沿 token 方案推进，可在不触碰业务逻辑的前提下快速完成主题化和视觉升级。
