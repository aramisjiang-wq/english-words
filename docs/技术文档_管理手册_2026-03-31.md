## 一、文档目的

本文档用于管理者快速掌握项目现状、上线地址、核心功能、配置策略、运维流程与后续管理要点，便于交接和持续运营。

## 二、项目基本信息

- **项目名称**: English Words（英语单词学习系统）
- **代码仓库**: [aramisjiang-wq/english-words](https://github.com/aramisjiang-wq/english-words)
- **线上地址**: [https://aramisjiang-wq.github.io/english-words/](https://aramisjiang-wq.github.io/english-words/)
- **部署方式**: GitHub Pages（`main` 分支根目录 `/`）
- **技术形态**: 纯静态站点（HTML + CSS + JS + JSON 数据）

## 三、当前版本核心能力

### 3.1 学习能力

- 单词浏览与状态标记（未学习/已掌握/难记）
- 多维筛选（分类、词性、主题、状态、预设）
- 测验、造句练习、听力练习、拼写练习
- 智能复习入口（基于复习数据）

### 3.2 数据与统计

- 学习历史（按天）
- 复习数据
- 成就系统
- 分析模块（热力图、趋势、点图）

### 3.3 设置与语音

- 支持语音选择与语速调整
- 默认语音策略已改为优先 `Google US English`

## 四、关键交互规则（已落地）

### 4.1 听力练习规则

- 默认不展示正确单词
- 用户先听音并输入答案
- 点击“检查答案”后才显示正确单词

### 4.2 批量操作规则

- 批量操作已独立为单独横条（`batch-bar`）
- 批量条为吸附式（滚动时保持可达）
- 批量按钮包含：
  - 批量选择
  - 批量掌握
  - 批量难记
  - 批量未学

### 4.3 页面入口去重

- 已移除重复功能入口
- 保留“学习/统计/设置”主导航
- 学习操作区聚焦在核心学习动作，避免同功能多处出现

## 五、视觉与排版改造要点

### 5.1 去 AI 味改造

- 全站主色从紫系调整为中性蓝/灰系
- 按钮、标签、图表、通知统一色彩策略
- 语义色保留（成功/警告/危险）

### 5.2 Banner 优化

- Header 由普通白底升级为轻量渐变
- 品牌区增加简洁徽标和更聚焦文案
- 信息层级：品牌 > 导航 > 统计

### 5.3 样式工程化

- 大量内联样式已迁移为 class
- 建立 token 化样式基础（`--color-*`）
- 会话页面模板已抽离（`session-renderers.js`）

## 六、语音默认策略（管理说明）

系统默认语音选择优先级如下：

1. 用户已保存语音（`localStorage.selectedVoice`）
2. `Google US English`
3. 任意 `en-US`
4. 任意 `en-*`
5. 浏览器默认语音

并监听 `speechSynthesis.onvoiceschanged` 处理异步加载。

## 七、数据与状态存储

当前状态主要存储在浏览器本地 `localStorage`：

- `wordStatus`
- `learningHistory`
- `wordReviewData`
- `achievements`
- `filterPreferences`
- `selectedVoice`
- `speechRate`

说明：
- 清理浏览器缓存会丢失本地学习记录
- 若需长期保存与多端同步，需后续接入后端存储

## 八、部署与发布管理

### 8.1 当前上线方式

- GitHub Pages from `main` branch `/`
- 每次 push 到 `main` 后自动触发 pages build

### 8.2 发布步骤（标准）

1. 本地修改并自测
2. 提交到 `main`
3. 等待 GitHub Pages 构建完成
4. 验证线上地址

### 8.3 快速检查清单

- 页面可访问
- 数据 `words_merged.json` 可加载
- 语音可播放
- 听力题默认不显示答案词
- 批量条吸附行为正常

## 九、常见问题与处理

### 9.1 本地 `8000` 打不开

- 需在项目根目录启动静态服务：
  - `python3 -m http.server 8000`

### 9.2 语音列表为空

- 某些浏览器语音列表异步加载，刷新或等待后重试
- 优先推荐 Chrome 测试

### 9.3 Pages 刚发布访问失败

- 构建可能仍在进行中，等待 1-3 分钟后刷新
